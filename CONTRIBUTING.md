# Contributing to OCR Finetuning

First off, thank you for considering contributing to OCR Finetuning! It's people like you that make the open-source community such a great place to learn, inspire, and create.

## Where Can I Help?

- **Reporting Bugs:** If you find a bug, please open an issue describing the problem and how to reproduce it.
- **Suggesting Enhancements:** Have an idea for a new feature? We'd love to hear it. Open an issue to discuss it.
- **Documentation:** Documentation improvements are always welcome. If you find something unclear, help us make it better!
- **Code:** If you want to contribute code, please follow the steps below.

## Development Setup

1. Fork the repository and clone it locally.
2. Follow the setup instructions in the [README.md](README.md).
3. Create a new branch for your feature or bugfix.
4. Ensure your code follows the existing style (TypeScript for backend, Vue.js for frontend).
5. Verify your changes by running the local development environment (`bash scripts/start_all.sh`).

## Pull Request Process

1. Ensure any install or build dependencies are removed before the end of the layer when doing a build.
2. Update the README.md with details of changes to the interface, this includes new environment variables, exposed ports, or documentation changes.
3. You may merge the Pull Request in once you have the sign-off of two other developers, or if you do not have permission to do that, you may request the maintainer to merge it for you.

## Code of Conduct

Please be respectful and professional in all interactions within the project community.

## Questions?

If you have any questions, feel free to open an issue or reach out to the maintainers.
